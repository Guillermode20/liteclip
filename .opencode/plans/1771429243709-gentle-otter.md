# Plan: Fix Console Windows Appearing for FFmpeg Processes

## Problem

On Windows release builds:
1. 3 console windows briefly appear at startup (encoder detection)
2. A console window appears each time a clip is saved (muxing)

## Root Cause

The code combines `CREATE_NO_WINDOW | DETACHED_PROCESS`:

```rust
const CREATE_NO_WINDOW: u32 = 0x08000000;
const DETACHED_PROCESS: u32 = 0x00000008;
pub const PROCESS_CREATION_FLAGS: u32 = CREATE_NO_WINDOW | DETACHED_PROCESS;
```

**According to Microsoft documentation:**

> CREATE_NO_WINDOW: This flag is **ignored** if used with DETACHED_PROCESS.

So `CREATE_NO_WINDOW` is being ignored, leaving only `DETACHED_PROCESS`. This means FFmpeg processes don't get the "no window" flag applied.

## Solution

Remove `DETACHED_PROCESS` and use only `CREATE_NO_WINDOW`:

```rust
pub const PROCESS_CREATION_FLAGS: u32 = CREATE_NO_WINDOW;
```

This will properly hide console windows for FFmpeg child processes.

## Files to Modify

**`src/encode/hw_encoder/functions.rs:11-14`**
- Change from: `CREATE_NO_WINDOW | DETACHED_PROCESS`
- Change to: `CREATE_NO_WINDOW`

## Verification

1. Build: `cargo build --release --features ffmpeg`
2. Run the application and verify:
   - No console windows appear at startup
   - No console window when saving a clip
3. Verify clips are saved correctly with both H.264 (NVENC/AMF/QSV) and MJPEG (software) encoders
