# RAM Optimization Plan for LiteClip Replay

## Completed Implementation

### Phase 1: Quick Wins ✓

#### 1.1 Reduced Channel Buffer Size
**File:** `src/capture/dxgi/types.rs:61`
- Changed from 256 → 16 frames
- **RAM saved:** ~1.9GB (256 × 8MB → 16 × 8MB)

#### 1.2 Removed Unused Texture Field
**Files:** `src/capture/mod.rs`, `src/capture/dxgi/types.rs`
- Removed `texture: D3D11Texture` from `CapturedFrame` struct
- Removed unused import and texture creation code

### Phase 2: Conditional CPU Readback ✓

#### 2.1 Added Capture Mode Flag
**File:** `src/capture/mod.rs`
- Added `perform_cpu_readback: bool` to `CaptureConfig`
- Wired to `config.advanced.use_cpu_readback`

#### 2.2 Implemented Conditional Skip
**File:** `src/capture/dxgi/types.rs:209-226`
- When `perform_cpu_readback = false`:
  - Acquire frame, get timestamp, release immediately
  - Return `CapturedFrame` with empty `bgra`
  - Skip GPU→CPU copy entirely (~8MB/frame saved)
