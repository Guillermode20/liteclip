# Fix: Clear Replay Buffer After Clip Save

## Problem
When recording multiple clips in succession, the second clip contains:
1. All the content from the first clip
2. Frozen frames (holding on last frame of old clip) for the remaining duration

## Root Cause
The `clear()` method exists in `ReplayBuffer` (`src/buffer/ring/types.rs:184-189`) but is **never called after a clip save**. The buffer retains all old frames, so subsequent clips include stale data with broken timestamp continuity.

## Solution
Call `buffer.clear()` after a successful clip save in `src/app.rs:save_clip()`.

## Implementation

### File: `src/app.rs`
Modify `save_clip()` (lines 221-253) to clear the buffer after successful save:

```rust
pub async fn save_clip(&self) -> Result<PathBuf> {
    // ... existing code ...
    
    let handle = spawn_clip_saver(buffer.clone(), duration, output_path.clone(), muxer_config);
    
    let result = handle.await.context("Clip saver task panicked")?;
    let final_path = result?;
    
    // Clear buffer after successful save
    self.buffer.clear();
    info!("Buffer cleared after clip save");
    
    Ok(final_path)
}
```

**Note:** Clone `buffer` before passing to `spawn_clip_saver` so we retain access to call `clear()` after.
